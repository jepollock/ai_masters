\section{Classification, Regression and Clustering}

These answers will deal with a single dataset, browser analytics events with backing system RPCs. The business has 10m users. ARPU is currently \(USD\ \$160/user/year\) \cite{arpu}. Total revenue is USD \$1.6billion. Each session will generate 100 clicks, and 100 front end RPCs. Each front-end RPC generates 100 back end calls. Users will interact with the system daily. To deal with seasonal variations in data, we need to store and process at least a full year's worth of data. 

Therefore the \textbf{volume} and \textbf{velocity} of data involved is roughly:

\begin{center}
	\begin{tabular}{|c c | c c|}
	\hline
	    \textbf{variable} & \textbf{description} & \textbf{variable} & \textbf{description}\\
	    \(S_d\) & sessions per day = 1 & \(C_s\) & clicks per session = 100\\
	    \(F_s\) & front-end RPCs per session = 100 & \(B_f\) & back-end RPCs per F\_s = 100\\
	    \(E_s\) & events per session & \(B_e\) & bytes per event = 100\\
	    \(B_s\) & bytes per session = 1000 & \(T_b\) & total data size\\
	    \(N_d\) & total number of days stored & \(T_e\) & total events stored analyzed per user\\
	\hline
	\end{tabular}
\end{center}

\begin{align*}
	E_s &= C_s + F_s + F_s * B_f\\
	T_e &= N_d * S_d * E_s\\
	&= N_d * S_d * (C_s + F_s + F_s * B_f)\\
	T_b &= (S_d * N_d * B_s + T_e * B_e) * U\\
	&= 3.73 \text{ petabytes per year}\\
	&= 946Mb/s.
\end{align*}

\section*{big data classification example}

 A model can be trained to predict if a user may be a fraudster, using the user event stream, including a "deemed fraudster" event. The "deemed fraudster" event would be removed from the stream and used as the label for training.

Since the dataset is labelled, this is a \textbf{classification} problem.

If the company cannot keep payment fraud under control, processors will increase merchant fees, or even disconnect the company from payment networks. The fraud loss ratio is currently 1.8\% of revenue \cite{ccfraud}, providing a \textbf{value} of USD \$32.4million.

Both fraudsters and the system will change. The model will need to be retrained frequently. If the model is retrained every month, and the data should be no more than 2 weeks old at launch, the system needs to process \(\frac{3.73PB}{2\ weeks} = 25Gbps\).

\section*{big data regression example}

The dataset includes a "purchase" event, with the cart value. We can label the user with the total spent over the past year. The goal is to predict a customer's total spend from the event stream. Since the dataset is labelled with a continuous value, it is a \textbf{regression} problem.

The ability to better target spending is of significant \textbf{value} to the business. A 1\% improvement in transaction completion is worth USD \$18million

\section*{big data clustering example}

The model should detect significant differences in sessions - too short, too long, or containing strange interactions.

There aren't any labels - this is a \textbf{clustering problem}. Every completed session's event stream will be evaluated against the model to determine if it is an anomaly, where "anomaly" is defined as "being too far away from any of the clusters".

The alerting system would count sessions reported anomalous by the model, and if they exceed a certain rate (or ratio), trigger an alert. The business \textbf{value} is in reducing revenue loss and escalations. Going from 3 9's to 5 9's of downtime would be worth USD \$2million in immediate revenue.

In Anomaly detection, we again want short training intervals, with a lag in the training data window to avoid boiling the frog as bad features are incrementally rolled out. This would still require training every month, requiring \(25Gbps\) of processing.

\section{Feature Selection/Construction Methods}

\subsection{feature ranking/feature weighting vs feature
	subset selection methods}
	
Feature Ranking is a global ordering of features, where each feature is evaluated once. This will then be used by a greedy subset search algorithm which will pick the top-K 
features \cite{week3fsfilter}.

Feature Subset Selection will iterate over subsets and evaluate each subset, selecting the best from those reachable using the algorithm.

Feature Ranking is very quick to perform typically \(O(N)\), but does not consider interactions between features. Feature Subset Selection can take significantly more time, a complete search would require \(2^N\) iterations.

Pearson Correlation Coefficient \cite{pearson1895note}, measures the linear relationship between a feature and a class label.

Information Gain, \cite{quinlan:induction} is derived from the entropy/purity/ratio of classes before and after a \{feature, value\} pair are used to split the dataset.

Feature Subset Selection is a search algorithm, producing candidate subsets for evaluation. Sequential Forward Search \cite{week2featmanip} grows the proposed feature set by adding the "best" feature from the remaining set. The algorithm completes when the set exceeds a preconfigured threshold.

Beam Search \cite{harpy}, grows N sets. At each iteration, the N sets are grown, with the best N results kept. This allows the search to spread out and test hypotheses.

\subsection{Define Filter, Wrapper, Embedded Feature Selection/Construction}

Filter feature selection/construction methods use a subset search without training a ML model.

Primary Component Analysis \cite{FRS1901LIIIOL} translates the features to a new set of orthogonal features using Eigenvectors. The features are sorted, and the top-K are selected.

Relief \cite{Kira1992TheFS} randomly picks values from the dataset, using Euclidean distance to find features that are able to find class boundaries.

Wrapper feature selection methods contain a subset selection algorithm and the wrapped machine learning algorithm. The target algorithm is used during feature selection.

Sequential Backwards Search \cite{week2featmanip} uses the same approach as Sequential Forward Search, beginning from a full set, and removing the "worst" feature at each stage.

Yu, L's paper \cite{subsetcover} recommends using set-cover to choose between forwards or backwards searches. The set cover provides a good hint at the final size of the feature set. 

Sequential Floating Forward Selection SFFS\cite{PUDIL19941119} alternates between adding and removing features from the set. This algorithm is not guaranteed to converge, so any implementations will need to keep track of already visited sets.

Embedded feature set selection/construction algorithms derive the feature subset during construction of the model.

CART DecisionTree\cite{breiman2017classification} can be used for both feature selection and feature construction. The feature list from the nodes of the tree is the feature subset. The boolean equations in the nodes can be considered new boolean features, allowing rows to be mapped onto the path through the tree.

Support Vector Machines \cite{Cortes1995SupportvectorN} finds the slope of a line between two classes that maximizes the distance between values in the class by mapping into a new set of dimensions.

\subsection{Compare and contrast the performance of supervised filter, wrapper, and embedded methods}

Filter methods generalize best to different problems, are generally cheap and provide the worst accuracy. Filter methods are cheap because they use cheap evaluation functions (linear). Filter methods do not evaluate the subsets in the target model, limiting the resulting accuracy. The subset's performance will be similar between models, allowing it to generalize better.

Wrapper methods generalize the worst, are very expensive but provide the best accuracy. Wrapper methods train an instance of the target model at least once per subset tested. Testing against the target model with subset search is more likely to find subsets containing features that work well together, but have limited individual predictive power. This gives wrapper methods high accuracy, but low generality. The feature subset is linked to the target model algorithm.
 
Embedded methods are in the middle, providing accuracy, cost and generality between the other two. Embedded methods have the same cost as Filter (both train the target model once), and are cheaper than Wrapper methods. Since Embedded only trains a single model, it will be cheaper than a Wrapper method which trains N models for each subset. Embedded methods perform feature selection during model construction.

\subsection{Correlation-based Feature Selection (CFS)1}

Yu, L. \cite{subsetcover} states that CFS is a Filter method. Reading Hall M.'s paper \cite{cfsHall}, CFS doesn't use a model, but does validate subsets instead of features individually. Since it doesn't use a model, it can't be embeded or wrapper, leaving filter the remaining class. CFS uses the BestMatch complete subset search algorithm, although Hall recommends adding an early exit criteria. Any complete search will produce similar results. Single forward search was also tested, but did not perform as well.
